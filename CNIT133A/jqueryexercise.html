Question: Is it possible to have multiple trigger types in HTMX?
Yes, it is. We add "hx-trigger" in the code, and we could use a comma to separate the "changes" and "load".
We need to display the "id=result" area, so we add the hx-target attribute.
The select code will be changed as follows:
<select name="city" id="city" hx-post="/api/search" hx-trigger="changes, load" hx-target="#result">
 In this scenario, the hx-post attribute specifies the "/api/search" to send the POST request to when the select item is "changes" or "load".
When the server gets "/api/search", it will be doing some process.
